# Fase 4: UI/UX Flow

## 1. Prinsip Desain
* **Mobile-First:** Menggunakan *Bottom Navigation Bar* (Beranda | Riwayat | Kendaraan | Profil).
* **Touch-Friendly:** Tombol besar yang mudah dijangkau ibu jari dan *layout* berbasis *Card*.

## 2. Alur Tampilan (Flow)
* **Halaman Beranda:** Menampilkan *dropdown* pemilih kendaraan, Odometer raksasa (*Hero Section*), status indikator kesehatan motor, daftar estimasi biaya, dan tombol melayang (FAB) "+" untuk tambah servis.
* **Halaman Input Data:** Menggunakan input *numpad* otomatis untuk angka, serta menyediakan *Chips/Tags* (tombol kecil) untuk pilihan *sparepart* instan (misal: [Oli Mesin], [Busi]) agar pengguna tidak perlu banyak mengetik manual.
* **Halaman Riwayat Servis:** Desain *timeline* vertikal yang menampilkan tanggal, total KM, ringkasan komponen, dan total biaya secara berurutan.
* **Halaman Saran:** Menggunakan *Bottom Sheet Modal* (layar kecil meluncur dari bawah) setelah menekan tombol simpan, berisi centang sukses dan pesan saran pintar.
